Isi dari planning nya adalah sebagai berikut:

buat table sessions:
- id (int, auto increment, primary key)
- user_id int not null (ini adalah foreign key dari tabel users)
- token varchar 255 not null (isinya UUID untuk token user yang login)
- created_at datetime default current timestamp
- expired_at datetime default current timestamp + 1 hour

buatkan API untuk registrasi login user

Endpoint : POST /api/users/login

Request Body : 

{
    "email": "john_doe@localhost",
    "password": "rahasia"
}

Response : 
{    
    "data": "token" -> ini adalah UUID yang di generate dari login
}

Response Error:
{
    "error": "Email atau password salah"
}
